# ADR-002: Интеграция через события (async events)

## Статус
Принято

## Контекст
Уведомления должны отправляться при изменении статусов заказов/платежей.
Нельзя жёстко связывать order и payment с notification.

## Решение
Используем брокер (Kafka) и события:
- OrderStatusChanged
- PaymentStatusChanged
- ProductViewed (для персонализации)
- ProductUpdated (для обновления read-моделей/индекса)

notification-service подписывается на события и отправляет уведомления.

## Причины
- Слабая связность сервисов.
- Удобно добавлять новых подписчиков (аналитика, антифрод) без изменения продюсеров.

## Последствия
- Eventual consistency.
- Нужна идемпотентность обработчиков и стратегия ретраев (consumer retry) — описано как follow-up.
