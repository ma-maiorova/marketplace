# ADR-001: Декомпозиция по доменам (domain microservices)

## Статус
Принято

## Контекст
Нужно спроектировать маркетплейс с доменами: Users, Catalog, Search, Feed, Orders/Cart, Payments, Notifications.
Требование ДЗ: описать архитектуру, показать связи данных (без shared DB), показать взаимодействия.

## Решение
Выделяем сервисы по доменным границам:
- user-service
- catalog-service
- feed-service
- order-service (включая cart)
- payment-service
- notification-service

## Причины
- Чёткие границы ответственности и владения данными.
- Независимое масштабирование: лента/поиск/каталог могут иметь другой профиль нагрузки, чем платежи.
- Упрощает объяснение критериев ДЗ (no shared DB, явные взаимодействия).

## Последствия
- Больше межсервисных взаимодействий.
- Потребуются контракты API событий, наблюдаемость и ретраи.
