# ADR-003: Отдельная БД на сервис (DB per service)

## Статус
Принято

## Контекст
Критерии ДЗ требуют отсутствия shared database между сервисами.

## Решение
Каждый сервис владеет своей БД (PostgreSQL). Другие сервисы получают данные только через API или события.
Search index и Redis рассматриваются как технические хранилища (кэш).

## Последствия
- Данные между сервисами могут дублироваться (read models).
- Нужна стратегия синхронизации (events), версионирование контрактов.
